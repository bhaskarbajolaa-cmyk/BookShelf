## 📖 About The Project

BookShelf is a robust backend microservice designed to handle real-time book searches and generate personalized reading recommendations. By leveraging low-level memory management in C++ and decoupling the data layer via Docker, this project demonstrates the real-world application of complex data structures in a modern web architecture.

### ✨ Core Algorithms (DAA)

> This API shifts heavy algorithmic processing from the database into in-memory C++ data structures for maximum performance.

* **Search Autocomplete (Prefix Tree / Trie):** Database catalogs are ingested into an in-memory Trie on server startup. This allows the `/api/search` endpoint to provide instant, case-insensitive autocomplete suggestions with a time complexity of **$O(L)$** (where *L* is the length of the search prefix).
* **Collaborative Filtering (Bipartite Graph BFS):** Recommendations are generated by performing a 2-hop breadth-first traversal across a User-Book interaction graph. The algorithm calculates frequency scores to find mathematically relevant recommendations based on overlapping user tastes.

## 🛠️ Tech Stack

* **Language:** Standard C++17
* **Web Framework:** [Crow](https://crowcpp.org/) (C++ Microframework)
* **Database:** PostgreSQL (interfaced via `libpqxx`)
* **Infrastructure:** Docker & Docker Compose
* **Data Ingestion:** Python (Google Books API)

You do not need to install a C++ compiler or PostgreSQL locally to run this project. The entire environment is containerized.

* [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running.
